# Story 1.4: Interações, Animações e Suporte a Teclado

## Status

Done

## Story

**As a** usuário final,
**I want** feedback visual satisfatório ao interagir com a calculadora e poder usar o teclado,
**so that** a experiência seja completa, acessível e alinhada com a personalidade pop art do projeto.

## Acceptance Criteria

1. Efeito de clique nos botões via CSS `:active`: `transform: translate(3px, 3px)` + `box-shadow` reduzida para `0px 0px 0px black` (simula afundamento físico)
2. Animação "POW!" implementada: elemento `.pow-animation` posicionado absolutamente sobre a calculadora, visível por 0.8s ao pressionar `=`, usando `@keyframes` CSS (escala de 0 a 1.2 e fade out)
3. Animação "POW!" adicionada via classe JS temporária (`.show-pow`) e removida após `animationend` event — não bloqueia input durante execução
4. Suporte a teclado implementado via `document.addEventListener('keydown')` com mapeamento:
   - `0–9` → `handleNumber(key)`
   - `.` → `handleDecimal()`
   - `+`, `-`, `*`, `/` → `handleOperator()` (com conversão: `*` → `×`, `/` → `÷`)
   - `Enter` ou `=` → `calculate()`
   - `Escape` → `handleClear()`
   - `Backspace` → remover último dígito do `currentValue` (bonus feature)
5. Tecla pressionada dispara o mesmo estado visual do clique: classe `.active` adicionada ao botão correspondente e removida após 100ms
6. Hover nos botões com `filter: brightness(1.1)` e `cursor: pointer`
7. Transição CSS suave nos botões: `transition: transform 0.1s, box-shadow 0.1s`
8. Testado em Chrome e Firefox: todas as teclas mapeadas funcionam corretamente
9. Animação "POW!" não interfere com operações rápidas consecutivas (múltiplos `=` em sequência — `animationend` garante remoção antes da próxima)
10. Página final sem erros no console do navegador

### Animação POW! — Lifecycle Completo

```
1. Usuário pressiona "=" ou Enter
2. calculate() executa
3. triggerPowAnimation() chamado
4. pow.classList.remove('show-pow') — reset estado anterior
5. void pow.offsetWidth — forçar reflow (reinicia animação CSS)
6. pow.classList.add('show-pow') — inicia animação 0.8s
7. animationend event — pow.classList.remove('show-pow')
8. Próximo "=" pode ser pressionado imediatamente (não bloqueado)
```

> **CRÍTICO:** Usar `animationend` com `{ once: true }` — não `setTimeout`. Isso garante que animações rápidas consecutivas não acumulam event listeners.

### CSS da Animação POW!

```css
.calculator {
  position: relative; /* necessário para o position: absolute do POW! */
  overflow: visible;
}

.pow-animation {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  display: none;
  font-family: 'Bangers', Impact, fantasy;
  font-size: 80px;
  color: var(--red);
  -webkit-text-stroke: 3px black;
  pointer-events: none; /* não intercepta cliques */
  z-index: 10;
  will-change: transform; /* GPU acceleration */
  text-align: center;
}

.pow-animation.show-pow {
  display: block;
  animation: pow-show 0.8s ease forwards;
}

@keyframes pow-show {
  0%   { transform: translate(-50%, -50%) scale(0) rotate(-10deg); opacity: 0; }
  60%  { transform: translate(-50%, -50%) scale(1.2) rotate(5deg); opacity: 1; }
  80%  { transform: translate(-50%, -50%) scale(1.0) rotate(-2deg); opacity: 1; }
  100% { transform: translate(-50%, -50%) scale(0.8) rotate(0deg); opacity: 0; }
}
```

> **Nota:** O `transform` nos keyframes deve manter o `translate(-50%, -50%)` para manter o elemento centralizado durante a animação.

### Keyboard Mapping Completo

```javascript
function handleKeyboard(e) {
  // Prevenir comportamentos padrão problemáticos
  if (e.key === '/') e.preventDefault(); // evita "quick find" em Firefox

  const keyMap = {
    '0': () => handleNumber('0'),
    '1': () => handleNumber('1'),
    '2': () => handleNumber('2'),
    '3': () => handleNumber('3'),
    '4': () => handleNumber('4'),
    '5': () => handleNumber('5'),
    '6': () => handleNumber('6'),
    '7': () => handleNumber('7'),
    '8': () => handleNumber('8'),
    '9': () => handleNumber('9'),
    '.': () => handleDecimal(),
    ',': () => handleDecimal(), // bonus: vírgula também funciona
    '+': () => handleOperator('+'),
    '-': () => handleOperator('-'),
    '*': () => handleOperator('×'), // conversão obrigatória
    '/': () => handleOperator('÷'), // conversão obrigatória
    'Enter': () => calculate(),
    '=': () => calculate(),
    'Escape': () => handleClear(),
    'Backspace': () => handleBackspace(),
  };

  const action = keyMap[e.key];
  if (action) {
    action();

    // Feedback visual: encontrar botão correspondente e piscar
    const dataValue = {
      '*': '×', '/': '÷', 'Enter': '=', 'Escape': 'AC', 'Backspace': null
    }[e.key] ?? e.key;

    if (dataValue) {
      const btn = document.querySelector(`[data-value="${dataValue}"]`);
      if (btn) {
        btn.classList.add('active');
        setTimeout(() => btn.classList.remove('active'), 100);
      }
    }
  }
}
```
